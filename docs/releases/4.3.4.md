# DeepSeek Infra 4.3.4 — Reload Transaction Integrity and Page-Lifecycle Recovery

<!-- docs-language-switcher:end -->

4.3.4 hardens the 4.3.3 update pipeline against interrupted, duplicated and half-applied reloads. Activation is one serialized transaction, update checks are bounded and supersedable, and every page exit path persists unsent Composer and conversation state.

## Serialized activation transactions

- Every build activation runs as a single-flight transaction: staging, consent, the synchronous persistence flush, Worker activation and the final reload are serialized, so concurrent checks or tabs cannot interleave or double-activate.
- Reload transaction integrity contracts cover the full phase machine, including blocked, activating and error recovery paths.
- Update deferral is phase-safe: deferring records the decision without leaving half-applied activation state, and the next check resumes from a clean store snapshot.

## Bounded, supersedable update checks

- Each update check is bounded by a timeout, so a stalled pointer fetch can never wedge the update pipeline in a checking state.
- A newer discovered target supersedes any pending result; a delayed or stale response can never resurrect a replaced build.

## Page-lifecycle persistence

- Composer drafts and conversation state flush synchronously on `pagehide`, `visibilitychange` and `beforeunload`, so closing, hiding or reloading a tab never loses unsent text.
- Composer drafts are scoped per conversation and per project in `sessionStorage`, isolating drafts between projects that share conversation views. The real Chromium smoke draft-key contract is aligned with the project-scoped keys.
- Message submission commits atomically through `tryStartMessage`/`peek`/`commit`: a submission either completes with its draft cleanup or leaves the draft untouched.

## Verification

- Frontend unit and contract tests cover activation transaction serialization, bounded/superseded update checks, phase-safe deferral, page-lifecycle flushes, project-scoped draft isolation and atomic message submission; the browser smoke aligns with the project-scoped draft keys.
