## Frontend Runtime Decomposition

4.1.0 scopes the frontend runtime by route. `main.tsx` now owns only the browser router. The ten workspace Providers mount around chat routes at `/` and `/ui/`, while `/trace/:traceId` renders without initializing chat, Projects, Skills, Memory, attachments, Activity, or Diagnostics state.

The Trace route is lazy-loaded behind `Suspense` and a route-level error boundary. Diagnostics also lazy-loads the shared `TraceDetailView`, keeping its selectors, visualizations, API normalization, and feature-owned CSS outside the initial workspace bundle until a user opens Trace functionality.

Trace requests now accept `AbortSignal` through the shared `HttpClient`. Route changes and component unmounts abort in-flight requests instead of merely suppressing late state updates.

## Executable contracts

`scripts/check_frontend_bundle.py` reads the Vite manifest and fails if the Trace page or detail view stops being a dynamic entry, if Trace implementation markers return to the initial bundle, or if Trace CSS is merged into the initial stylesheet.

The Chromium release smoke additionally verifies that a direct Trace route makes no Workspace API requests, that Trace chunks are absent before navigation and loaded afterward, and that deep-link refresh plus offline application-shell behavior remain intact.

TanStack Query, broad Context rewrites, and application-wide CSS decomposition remain outside this release.
