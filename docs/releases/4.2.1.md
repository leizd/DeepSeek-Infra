# DeepSeek Infra 4.2.1

<!-- docs-language-switcher:end -->


4.2.1 is a reliability patch over the 4.2.0 Query foundation: it hardens the interaction layer of the Query workbench instead of migrating more state.

## Included

- Memory drawer opens fetch exactly once; reopening follows Query staleness instead of a manual refresh loop.
- Stale `activeProjectId` restored from `localStorage` is repaired automatically when the project no longer exists.
- Project skill bindings use a dedicated Query hook (error + retry) and scope-serialized save mutations; late responses from a previously selected project cannot overwrite the current one.
- All three drawers show a non-blocking sync indicator during background refetch, an error block with retry, and per-mutation button disabling.
- Six new browser contracts cover the Query interaction behaviors; the initial bundle now has a 450 KB budget check.

## Deliberately not included

- No ChatContext migration, no attachment upload changes, no Query Devtools in production, no offline Query persistence, no global CSS rewrite, no backend protocol changes.
