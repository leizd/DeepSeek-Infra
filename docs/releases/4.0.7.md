4.0.7 switches the default entry to the React frontend after the parity releases (4.0.4–4.0.6), with a one-variable rollback.

## Included

- `/` serves the React build by default; `/legacy` serves the complete legacy workspace; `DEEPSEEK_FRONTEND=legacy` restores the previous default without a redeploy.
- Frontend-aware PWA assets: React mode serves a root-scoped runtime service worker and a manifest with `share_target` at the root paths; legacy mode keeps the original `sw.js` and manifest.
- Missing React build output falls back to the legacy index, so a fresh clone without `npm run build --prefix frontend` still starts.
- Auth token redirect flow covers `/legacy`; the Android Gradle build runs the frontend build before packaging (best-effort).
- Docker, PyInstaller, release archive, and Android packaging all verified to ship `static/ui/`.

## Deliberately not included

- No legacy file is deleted (`static/index.html`, `static/app.js`, `static/modules/chat.js` remain); `refactor(frontend): retire legacy chat runtime` ships only after the switched default proves itself.
- The frozen 4.0 runtime/protocol contract, Python-first ownership, Rust opt-in delegates, and Python fallback are unchanged.
