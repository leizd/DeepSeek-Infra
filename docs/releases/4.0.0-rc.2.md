# DeepSeek Infra 4.0.0-rc.2

`4.0.0-rc.2` supersedes `v4.0.0-rc.1` because the earlier candidate was frozen before the verified 3.5-3.10 hybrid-runtime work landed. rc.1 remains a historical architecture preview and must not be promoted directly to stable.

## Freeze scope

This candidate refreshes 4.0 from the verified 3.10.0 baseline and carries forward Gateway and MCP preparation, Tool Policy, RAG ranking and document preparation, compact vector transport, semantic-cache dual storage, release performance evidence, and the complete 3.5-3.10 product evidence surface.

The architecture is Python-first: Python remains default and authoritative; default Compose is Python-only; the Rust sidecar is supported but optional; every delegate is default-off; and Python fallback remains supported throughout 4.x. Python retains Gateway streaming, upstream HTTP/credentials/retries, MCP transport/session/tool execution, files/OCR/embedding/SQLite/index ownership. Rust-primary ranking is not enabled.

## New release gates

- Rust 1.85 workspace line coverage is 80.4329% (3716/4620) with `cargo-llvm-cov 0.6.21`, blocking below 80% across all five core crates.
- Python CI remains at 95%; two complete rc.2 runs each measured 95.2317%, above the 95.20% freeze minimum.
- Upgrade from 3.10.0 and rc.1, rollback to 3.10.0, Python-only startup, and sidecar-loss fallback are executable contracts.
- The ten supported Rust endpoints, schemas, media types, limits, stable errors, ownership, fallback owner, and stability class are frozen. `DSVRNK01` and `DSVRSP01` are unchanged.
- Parity is complete at Gateway 68, MCP 105, RAG 38, RAG document 125, and vector binary 110 valid plus 16 malformed.
- The release dry-run produces the rc.2 ZIP, checksum, and rich manifest bound to the exact commit.

## Non-goals

This candidate does not enable or deploy Rust by default; enable Rust-primary ranking; sample parity; delete JSON embeddings or Python fallback; move Gateway streaming/upstream HTTP, MCP transport/tool execution, or file/OCR/embedding/SQLite/index ownership to Rust; break existing APIs; publish stable `4.0.0`; create a tag; or create a GitHub Release.

Stable `4.0.0` requires an independent promotion PR after rc.2 observation.
