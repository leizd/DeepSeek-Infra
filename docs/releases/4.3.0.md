# DeepSeek Infra 4.3.0 — Workspace Demand Loading

<!-- docs-language-switcher:end -->

4.3.0 removes optional Workspace UI, Provider and list-query work from the chat cold path without changing backend protocols or the frozen 4.0 runtime contract.

## What changed

- One typed feature registry owns Settings, Projects, Skills, Memory, Reminders, Diagnostics, file preview, image lightbox and Activity imports.
- Pointer, focus and touch intent deduplicate code preload. Preload does not open UI and does not start Skills or Memory list queries.
- The overlay host mounts only the active drawer. Projects and Skills demand-mount their shared Skills runtime; Memory keeps write support at the Chat root but demand-mounts its list provider.
- Feature-owned CSS is split from the initial stylesheet. Optional import failures stay inside a local retry/close boundary.
- The build emits `workspace-assets.json`. The Service Worker precaches the core shell, warms optional assets after load and retains current plus previous build caches for offline upgrades.

## Release gates

`python scripts/check_frontend_bundle.py` enforces:

- initial entry <= 390,000 bytes;
- at least 8% smaller than the 4.2.8 entry baseline of 425,914 bytes;
- initial CSS <= 28,000 bytes;
- every optional JavaScript chunk <= 90,000 bytes;
- dynamic Workspace entries, feature-owned CSS and a resolvable offline asset inventory.

`python scripts/smoke_frontend_browser.py` uses real Chromium to verify cold-load deferral, intent preload/query separation, demand query start, repeat-open reuse, latest-drawer wins, chunk failure/retry, Mutation continuity and offline optional reopen.

## Compatibility and release trust

- Chat-critical Providers, Query/Mutation continuity and existing 4.2.x write reliability stay intact.
- No frontend dependency, backend protocol, credential handling, Python/Rust ownership or fallback contract changes.
- Formal evidence is produced only by the clean exact-merge CI chain introduced in 4.2.8; dirty-worktree runs are development validation, not GA proof.

## Reproduce locally

```powershell
npm run check --prefix frontend
python scripts/smoke_frontend_browser.py
python scripts/preflight_release.py --version 4.3.0
```
