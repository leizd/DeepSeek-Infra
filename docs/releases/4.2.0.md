# DeepSeek Infra 4.2.0

<!-- docs-language-switcher:start -->
[中文](../../README.md) / [English](../../README.en.md)
<!-- docs-language-switcher:end -->


4.2.0 migrates the three server-resource domains of the React workbench onto a shared TanStack Query foundation, without touching any chat-side context.

## Included

- Projects, Skills and Memory controllers rewritten as `useQuery` + `useMutation` over a shared QueryClient with centralized query keys.
- AbortSignal-cancellable list requests threaded through `HttpClient` to the underlying fetch.
- `refreshing` vs `loading` semantics, query/mutation-derived error state, and `setQueryData` updates with background invalidation for create/rename/delete/toggle/remove/clear.
- Per-project skill-binding caches keyed by projectId with targeted invalidation.
- renderHook behavior tests (isolated QueryClient per case, retries disabled) covering initial queries, activation on create, cache updates, targeted invalidation, upload refresh, binding isolation, and 409 cache purity.
- Chat, Attachments, Activity, Diagnostics, Settings and Overlay contexts unchanged.

## Deliberately not included

- No changes to the frozen 4.0 runtime/protocol contract, Python-first ownership, Rust opt-in delegates, or Python fallback.
- No further Context migrations; Attachments' XHR upload queue is intentionally left outside Query.
