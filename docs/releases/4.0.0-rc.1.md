`4.0.0-rc.1` is a prerelease candidate for testing the readiness-approved 4.0 architecture and release artifacts. It is not recommended as a direct replacement for the latest stable production release.

## Runtime architecture

- Python remains the default runtime.
- Rust Gateway, MCP, Policy, and RAG delegates remain opt-in.
- The default Compose deployment remains Python-only.
- The Rust sidecar remains an optional deployment.
- Gateway streaming remains Python-owned for 4.0.
- Real MCP tool execution remains owned by the Python Tool Runtime.
- Python fallback remains supported throughout the 4.x release line and cannot be considered for removal before 5.0.0.

These boundaries are governed by [ADR-0040](../adr/ADR-0040-hybrid-runtime-architecture.md). This release candidate does not change that decision.

## Quality and evidence

- Python combined statement-and-branch coverage gate: 95%.
- Consecutive readiness runs before the freeze: 95.3428% and 95.3396%.
- Rust formatting, Clippy with warnings denied, workspace tests, Docker smoke, hybrid delegation/fallback, and 38/38 RAG parity remain release gates.
- Release smoke, offline evaluations, security corpus, Agent Eval, preflight, doctor, documentation links, and strict RC readiness are required to pass.
- The source archive, SHA-256 checksum, and manifest are generated from the frozen RC branch without committing the archive into Git history.

## Testing guidance

Testing the optional Rust sidecar and hybrid fallback is encouraged. Keep all Rust flags at `0` to verify the default Python-only path, then enable delegates individually to test opt-in behavior and fallback.

When reporting an issue, include the relevant readiness or evidence artifact, the release manifest, operating system, Python version, enabled Rust flags, and reproduction steps.

## Release posture

This pull request prepares and freezes the release candidate. It does not create the `v4.0.0-rc.1` Git tag. After the PR is merged and CI is green on the exact merge commit, release engineering may create the annotated tag and publish a GitHub prerelease with the archive, checksum, manifest, and these notes.
