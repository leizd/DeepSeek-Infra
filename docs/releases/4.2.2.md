4.2.2 closes the 4.2.x Query series: failed operations recover cleanly, and every write lands in the Query cache immediately.

## Included

- Stale mutation errors clear via newest-submittedAt selection plus a `recover()` that resets failed mutations and refetches (drawer action: 重新同步）.
- Binding save retries replay the last desired state; binding mutation state is isolated per project via `key={project.id}`.
- Chat memory suggestions save through the Memory Query cache — an open drawer updates instantly, conflicts still surface for replacement.
- Legacy skill-binding controller API removed; `useProjectSkillBinding` is the single binding path.
- Query retry policy: AbortError and plain 4xx never retry; 408/425/429/5xx and network errors retry once.
- Six new browser contracts and hook tests covering recovery, replay, isolation, cache coherence, and retry rules.

## Deliberately not included

- No ChatContext Query migration, no attachment upload changes, no offline Query persistence, no Context removal, no drawer lazy loading (planned as 4.3.0), no backend protocol changes.
