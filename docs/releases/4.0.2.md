4.0.2 establishes the React migration boundary without replacing the stable frontend.

## Included

- A separate React 19 + TypeScript + Vite source tree in `frontend/`, built to the gitignored `static/ui/` production tree.
- A stable legacy workspace at `/` and an isolated React preview at `/ui/`, including SPA deep-link fallback.
- Pure TypeScript chat and conversation types, an immutable stream-event reducer, a shared HTTP client with normalized API errors, and an async NDJSON iterator.
- Vitest coverage for stream decoding, chunk boundaries, reducer transitions, interruption, errors, authentication injection, and unknown-event compatibility.
- Required frontend typecheck, tests, build, npm audit, browser smoke, Docker build, PyInstaller build, and release-package integration.

## Deliberately not included

- `/` does not switch to React in this release.
- React and legacy JavaScript never manage the same DOM tree.
- TanStack Query, a global React store, Composer, message rendering, Projects, Skills, Memory, Settings, Speech, and PWA ownership remain future migration slices.
- The frozen 4.0 runtime/protocol contract, Python-first ownership, Rust opt-in delegates, and Python fallback are unchanged.
