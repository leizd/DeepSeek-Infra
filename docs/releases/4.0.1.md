4.0.1 is a frontend-security and offline-reliability patch. It does not change the frozen 4.0 protocol, Python-first runtime ownership, default Compose behavior, Rust opt-in boundaries, or Python fallback contract.

## What changed

- The first-paint theme bootstrap is now a same-origin script and Inter is vendored locally with its OFL license. The strict `script-src 'self'` and `font-src 'self'` CSP remains intact.
- DeepSeek and Tavily keys are no longer persisted in `localStorage`. Remembered values are scoped to the current tab session, and legacy plaintext local keys are purged.
- Uploads expose cancellation, enforce a four-minute default timeout, handle abort and timeout explicitly, and restore UI state through a shared `finally` path.
- The Service Worker now caches the complete versioned CSS, JavaScript, font, and application shell; query strings are normalized; optional assets cannot fail installation; and static resources use stale-while-revalidate.
- Workspace navigation is implemented as real ARIA tabs with click, arrow-key, Home, and End behavior.
- Credential, upload, Workspace tab, and Skill Builder responsibilities moved into focused native ES modules while `chat.js` remains the orchestration entry point.
- A dedicated Playwright/Chromium CI job verifies CSP, first-paint theme, mock chat, upload cancellation, complete offline resources, and offline reload.

## Verification

```bash
python scripts/smoke_release.py --offline
python scripts/smoke_frontend_browser.py --out docs/evidence/frontend-browser-v4.0.1.json
python scripts/preflight_release.py --version 4.0.1 --ga
```

The browser dependency is isolated in `requirements-browser.txt`; the application remains vanilla JavaScript with no bundler, build step, or `package.json`.
