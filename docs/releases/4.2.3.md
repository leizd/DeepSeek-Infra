# DeepSeek Infra 4.2.3

<!-- docs-language-switcher:start -->
[中文](../../README.md) / [English](../../README.en.md)
<!-- docs-language-switcher:end -->


4.2.3 is a focused post-Query reliability patch: every failed workspace action stays inside the UI, every concurrent mutation keeps its own pending state, and duplicate same-entity writes are suppressed.

## Included

- Shared `runUiAction` dispatch containment for Project, Skill, Memory and project-binding actions, including recovery failures.
- Central mutation keys plus MutationCache-backed `isPending` methods for concurrent Project, Skill and Memory operations.
- Synchronous keyed action locks for entity writes; rapid duplicate dispatch sends one backend request.
- Failed project rename drafts remain editable; Project and custom Skill deletion require confirmation.
- Cache-writing mutations cancel stale reads before committing and invalidate after settlement.
- Hook and component contracts plus eight Chromium checks, including concurrent Memory deletion and stale-list cancellation.

## Deliberately not included

- No Provider lifecycle changes, ChatContext migration, attachment upload migration, offline Query persistence, drawer lazy loading, bundle-budget increase, visual redesign, or backend protocol changes.
