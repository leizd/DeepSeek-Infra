# DeepSeek Infra 4.0.0

`4.0.0` is the stable, metadata-only promotion of the validated `4.0.0-rc.2` candidate. The promotion changes the release identity and publication state; it does not change runtime behavior, protocol schemas, deployment defaults, or architecture ownership.

`v4.0.0-rc.1` remains a historical architecture preview. It was superseded because it predated the verified 3.5-3.10 hybrid-runtime work. Stable 4.0.0 is promoted only from rc.2 after its observation period and through this independent promotion change.

## Stable architecture contract

- Python remains the default and authoritative runtime, and default Compose remains Python-only.
- The Rust sidecar is officially supported but optional. Every Rust delegate and compact binary vector transport remain disabled by default.
- Python fallback remains supported throughout 4.x.
- Python continues to own Gateway streaming, upstream HTTP, credentials, retries, MCP transport/session/tool execution, files, OCR, embeddings, SQLite, indexes, retrieval, and business state.
- Rust request preparation, protocol preparation, policy evaluation, vector ranking, and document preparation remain explicit opt-ins.
- Rust-primary ranking is not enabled.

## Validated release gates

- Python coverage: two complete rc.2 qualification runs measured 95.2317%; the stable promotion keeps the blocking 95% CI gate and requires exact-promotion evidence at or above 95.20%.
- Rust coverage: 80.22% (3225/4020 lines) across 172 workspace tests in the stable promotion run, with an 80% blocking gate.
- Parity: Gateway 68/68, MCP 105/105, RAG 38/38, RAG document preparation 125/125, and vector binary 110 valid plus 16 malformed.
- Upgrade/rollback: 3.10.0 and rc.1 upgrades, rc.2-to-stable promotion, rollback to 3.10.0, Python-only startup, and sidecar-loss fallback are executable contracts.
- Protocol: the ten supported Rust endpoints, schemas, media types, payload limits, stable errors, fallback owners, business owners, and stability classes remain frozen. `DSVRNK01` and `DSVRSP01` are unchanged.

## Publication artifacts

The ZIP, checksum, manifest, Rust sidecar identity, and Git tag are regenerated and verified from the exact merged stable-promotion commit. The GitHub Release is a non-prerelease stable release and must not reuse rc.2 or PR-head artifacts.

## Non-goals

This release does not enable or deploy Rust by default; enable Rust-primary ranking; sample parity; delete JSON embeddings or Python fallback; move Gateway streaming/upstream HTTP, MCP transport/tool execution, or file/OCR/embedding/SQLite/index ownership to Rust; change the frozen protocol contract; break existing APIs; or create a new feature milestone.
