## React Trace Viewer

4.0.9 moves the last standalone application UI into the React frontend. `/trace/:traceId` now renders a routed Trace workspace from the same Vite build as `/`, while the authenticated backend route preserves the existing not-found behavior for unknown trace IDs.

The page includes trace metadata, duration/token/cache summary, parent-aware span tree, timing waterfall, category aggregation, error reporting, and JSON export. The Diagnostics drawer reuses the same loader, summary, and waterfall components instead of maintaining a second rendering path.

The obsolete `static/trace_viewer.html`, `static/modules/trace_viewer.js`, and `static/modules/trace_waterfall.js` files are removed. Static icons, Inter fonts, and KaTeX vendor assets remain unchanged.

## Routing and tests

`react-router-dom` now owns `/`, `/ui/`, `/trace/:traceId`, and the not-found route. FastAPI continues to return the React index for extensionless SPA paths. React Testing Library, `user-event`, and scoped jsdom tests cover routed Trace rendering, failed loads, selectors, and navigation without changing the Node environment used by reducer and API tests.

The Chromium release smoke opens a real persisted Trace, verifies the React waterfall, reloads the deep link, and rejects any return of the legacy Trace script.
