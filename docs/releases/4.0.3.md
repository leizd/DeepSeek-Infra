# DeepSeek Infra 4.0.3

4.0.3 moves the first complete user workflow into the isolated React application without switching the default frontend.

## Included

- Normal chat at `/ui/` with typed request construction, NDJSON streaming, Markdown rendering, model/thinking/search controls, title generation, and stop-generation.
- Reducer-driven conversations, migration from compatible legacy browser history, local history restore/new/delete flows, and separate Chat, Settings, and Overlay contexts.
- Memory-only DeepSeek and Tavily credentials. No credential is written to local storage, conversation history, logs, tests, or release evidence.
- Stream cleanup that cancels unfinished readers and always releases their lock; normal chat completion no longer sets a synthetic Agent-run status.
- Vitest coverage for reducers, request payloads, persistence migration, Markdown, stream decoding, interruption, and cleanup.
- Chromium coverage for React send, reload persistence, stop-generation, SPA fallback, legacy CSP/upload/offline behavior, and console safety.
- PyInstaller packaging that includes the required `multipart` module and fails early when the incompatible `python-multipart` distribution contaminates the build environment.

## Deliberately not included

- `/` does not switch to React and `static/modules/chat.js` is not deleted.
- Attachments, Agent/activity, Projects, Skills, Memory, advanced settings, speech, diagnostics, and PWA ownership remain on the legacy frontend until later parity slices.
- The frozen 4.0 runtime/protocol contract, Python-first ownership, Rust opt-in delegates, and Python fallback are unchanged.
