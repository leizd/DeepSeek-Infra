# DeepSeek Infra 介绍 PPT · 设计系统（_design.md）

> 所有页面作者必须严格遵守。场景：技术与工程类项目介绍；读者为工程师与技术决策者。
> 尺寸 960×540（16:9）。禁止：卡片墙（圆角矩形堆叠做层级）、蓝白/蓝紫渐变、玻璃拟态、彩虹多色、三等分/2×2 公式化布局、emoji。

## 1. 色彩（全部通过 theme.colors 引用，用 $token）

| token | 值 | 语义 |
|---|---|---|
| `$bg` | `#F2F1ED` | 页面底色：浅暖灰（编辑感、克制） |
| `$ink` | `#082D4F` | 正文与标题主色：深海军蓝 |
| `$accent` | `#D27347` | 暖陶土：kicker、关键路径、强调数字、当前态 |
| `$accentStrong` | `#F05023` | 信号橙：极少量用于最关键数字/关键路径加粗，一页至多一处 |
| `$structure` | `#3E4A5F` | 石板蓝：结构色——架构节点、模块、线条、坐标轴 |
| `$gray` | `#7A7F87` | 次要文字、注释、来源标注 |
| `$hairline` | `#D8D6D0` | 细分割线、表格线、网格线 |
| `$panel` | `#FFFFFF` | 直角面板底（仅用于代码/配置/关键结论/系统边界） |
| `$softCool` | `#E2E8EE` | 石板蓝浅底：结构分组填充 |
| `$softWarm` | `#F5E4D8` | 陶土浅底：强调分组填充 |
| `$ok` | `#3E7A5E` | 状态色：通过/Working（克制使用，允许表格内状态列） |

语义铁律：**暖色（accent 系）= 标题线索、关键路径、强调；冷色（structure 系）= 结构、节点、中性信息**。全 deck 一致。

## 2. 字体

| 角色 | fontFamily | 用途 |
|---|---|---|
| 展示标题 | `思源宋体` | 封面主标题、章节数字、超大数字、结尾语 |
| 页面标题/正文/标签 | `MiSans` | 一切功能性文字 |
| 代码/命令/指标值 | `MiSans`（配 `$structure` 色 + 适当 letterSpacing） | 保持字族克制，全 deck 不超过两个字族 |

字号阶梯：页标题 26 bold；kicker 标签 11（letterSpacing 2）；正文 13–14（lineHeight 1.5–1.6）；注释/来源 10–11 `$gray`；超大展示数字 56–120（思源宋体）。

## 3. 页面骨架（内容页统一）

- 顶部文档轴：y=0..40。左侧 `DEEPSEEK INFRA`（10px, MiSans bold, letterSpacing 3, `$structure`，x=48）；右侧页眉标签（10px, `$gray`，右对齐到 x=912）。y=44 处 1px `$hairline` 横线（x 48→912）。
- kicker：y=60，11px `$accent`，letterSpacing 2，形如 `01 · 架构总览`。
- 页标题：y=80 起，26px MiSans bold `$ink`，一句话结论式标题（陈述句，不给空泛名词）。
- 主内容区：y≈130..500。页脚右下角页码可并入顶部右侧标签（如 `架构 · 04 / 13`）。
- 安全边距：左右 48，上 44（轴线以上），下 28。
- 封面与结尾页不受顶部轴约束，但保留网格肌理与斜切符号。

## 4. 签名工艺元素（borrowed craft，全 deck 复用）

1. **网格肌理**：封面、结尾页背景叠加 40px 间距细网格（横竖 1px 线，色 `#082D4F0D`）。内容页不用。
2. **斜切符号（slash motif）**：一个陶土色平行四边形 `shapeName: parallelogram`（adjustments 默认即可），作为视觉签名——封面右侧一道大斜切（贯穿页高），章节/结尾处小斜切呼应。全 deck 只此一种装饰形状，直角优先。
3. **超大展示数字**：关键数字页用思源宋体 64–110px 数字 + 11px 标签，靠字号对比而非卡片分区。
4. **文档轴 + 1px 底线**：每页固定，营造"工程文档"感。

## 5. 图形与图表纪律

- 架构图：直角嵌套区域（rect，1px `$structure` 边框或 `$softCool` 浅底），箭头用 line 元素带 `arrow: [null, arrow]`；**关键路径用 `$accent` 2.5px，其余 1px `$structure`**；调用方向、协议名直接标注在线旁。
- 分层架构：层与层之间标注"拥有/委托/外部"边界；可选 Rust Sidecar 用虚线边框（border style: dash）表示"默认禁用、可回退"。
- 模块全景：hub-spoke——中心 FastAPI 核心（直角 rect `$ink` 底白字），13+ 模块用细线连出，模块名+编号直接标在线端，不用卡片。
- 流程链路：横向节点链，节点=直角 rect（`$panel` 底 + 1px `$structure`），关键路径节点边框/连线改 `$accent`；分支（缓存命中、工具循环最多 5 轮）用 dash 线。
- 图表：去框、淡网格线 `$hairline`，主系列 `$structure` 或 `$accent`，基线/次要系列灰；测试环境/版本/样本紧贴图表注明（10px `$gray`）。
- 表格：细线 `$hairline`、无厚外框、表头 `$softCool` 浅底；状态列固定；数字右对齐。
- 截图：直角矩形原样裁切（不圆角、不加阴影设备框），等宽等高并排，附 10px 说明。
- icon 如需使用：仅 fas 线性风格，单色 `$structure`，只作识别不作装饰。

## 6. 节奏

13 页：封面 → 定位（文字） → 关键数字（大字报） → 架构分层（图） → 请求链路（图） → 模块全景（图） → 安全（图+表） → 质量证据（图表） → 界面实录（截图） → 多端部署（表） → 版本演进（时间线） → 路线图与边界 → 结尾（回到决策与下一步，禁止单独 "Thank you"）。
